# Polaris UI 重设计方案：学习全球主流 Agent 设计

**日期**：2026-07-23  
**当前问题**：Polaris 的 UI 启文风格 + 不断叠加新功能，变得杂乱
- 顶部 4 行（logo + token条 + 按钮 + 窗口控制）
- 侧边栏：对话列表 + 模板库 + 工具/插件面板
- 底部 3 行（策略选择 + 联网/模板开关 + 提示文字 + 输入框）
- 聊天区：消息行包含意图标签、模型路由、操作按钮

**目标**：整理出清晰的信息架构，参考最优秀的 AI Agent 设计

---

## 一、值得研究的 5 个标杆项目

### 1. Cursor IDE（最完美的 Agent IDE 设计）

**GitHub**：闭源，但设计语言公开  
**核心设计理念**：

```
┌────────────────────────────────────────────────────────┐
│  Tab 行: [main.ts] [App.tsx] [Settings.tsx]            │
├────────────────────────────────────────────────────────┤
│                    编辑器主区                            │
│                                                        │
│                                                        │
├─────────────────────┬──────────────────────────────────┤
│  Ctrl+L 唤起        │                                  │
│  ⌘ 对话面板          │      Agent 输出区                 │
│  · 简洁气泡          │      · 代码 diff 卡片             │
│  · 流式逐字          │      · Accept/Reject 按钮         │
│  · 多文件上下文       │      · 终端输出                   │
│  · 模型选择器（小）   │                                  │
└─────────────────────┴──────────────────────────────────┘
```

**Polaris 可以学的**：
- **Composer 面板**：Ctrl+L 呼出内联对话，不占全屏。Polaris 应该有个"小窗模式"——不是每次对话都占满整个窗口
- **代码 Diff 卡片**：Agent 修改代码后，以绿色+红色的 diff 呈现，不是纯文本
- **Accept/Reject**：每个 Agent 操作都可以接受或撤销，不是一次性输出

### 2. Raycast AI（效率工具的黄金标准）

**设计理念**：浮动窗口 + 命令栏式交互

```
┌──────────────────────┐
│  搜索框（全局入口）     │  ← 全局快捷键唤起
│  ──────────────────── │
│  快捷操作:             │
│  · AI Chat     ⌘⏎    │
│  · 搜索文件     ⌘F    │
│  · 剪贴板历史  ⌘V    │
│  · 窗口管理     ⌘W    │
│  ──────────────────── │
│  AI 对话区域:          │
│  [问题]               │
│  [流式回答]           │
│  [复制 · 重新生成]     │
└──────────────────────┘
    浮动小窗，不占全屏
    15% 命令 / 85% 对话
```

**Polaris 可以学的**：
- **全局快捷键唤起后是浮窗，不是全屏**。Polaris 也有 Ctrl+Shift+Space 快捷键，但唤起的是大窗口。应该学习 Raycast 的小窗模式
- **命令即入口**：输入框不只是聊天，也是一个命令中心。输入"/"触发快捷操作
- **无边框极简**：Raycast 几乎没有可见的 UI 控件——只有输入框和结果

### 3. Claude Desktop / ChatGPT Desktop（大厂标准）

**设计理念**：对话流 + 侧边栏 + 极简控件

```
┌────────────┬──────────────────────────────────────────┐
│ 侧边栏      │  对话区                                   │
│            │                                          │
│ [+新对话]   │  [用户消息 1]                             │
│            │  [AI 回答 1]                              │
│ 对话 1      │                                          │
│ 对话 2      │  [用户消息 2]                             │
│ 对话 3      │  [AI 回答 2]                              │
│            │                                          │
│            │  ────────────────────────────             │
│            │  [输入框                        ] [发送]   │
└────────────┴──────────────────────────────────────────┘
```

**通用特点**：
- 对话列表在左侧，历史可回看
- 回答区是 Markdown 流式渲染，干净整洁
- 顶部极简（只有新建按钮 + 标题）
- 底部只有输入框 + 发送按钮（没有多余控件）
- 所有设置藏在侧边栏或独立面板

**Polaris 可以学的**：
- 当前 Polaris 顶部和底部太多控件了（4 行顶栏 + 3 行底栏）
- 应该把"策略选择""联网开关""模板"这些藏进一个弹出菜单或侧边栏
- 输入框应该是绝对的主视觉焦点——其他都是次要的

### 4. MiniMax OpenRoom（浏览器桌面 Agent）

**GitHub**：https://github.com/MiniMax-AI/OpenRoom  
**设计理念**：Agent 在聊天中直接展示操作过程

```
对话流:
  用户: 帮我把下载的 PDF 整理一下
  
  Polaris: [正在扫描 Downloads 文件夹...]
          · 找到 12 个 PDF
          · 按日期分类: 2024-01 (3个), 2024-02 (5个)...
          [正在移动文件...]
          ✓ 完成! 12 个文件已整理到 Documents/PDF/
  
  用户: 打开 Chrome 搜一下今天的新闻
  
  Polaris: [打开 Chrome ✓]
          [搜索 "今天的新闻" ✓]
          这是搜索结果摘要...
```

**Polaris 可以学的**：
- Agent 的**操作过程应该可见**。不是只给最终答案，而是展示"正在做什么"
- 每步操作是一个小卡片，有 ✓ ✗ 状态
- 这个其实跟 Polaris 的 Planner 和 Workflow 天然匹配

### 5. CodeConductor（Electron AI Agent 桌面壳）

**GitHub**：https://github.com/zhu1090093659/CodeConductor  
**设计理念**：极简桌面壳，对话驱动

特点：
- 纯命令行 + 对话混合界面
- 支持多个 AI CLI（Claude Code / Codex / Qwen）
- 专注功能而非花哨 UI

## 二、Polaris 当前 UI 的问题诊断

| 问题 | 具体表现 | 严重度 |
|------|---------|--------|
| **信息过载** | 顶部 4 行控件、底部 3 行控件 | 🔴 严重 |
| **层级不清** | 意图标签、路由信息、模型名全部平铺在消息中 | 🔴 严重 |
| **缺少焦点** | 输入框被各种按钮包围，视觉权重不足 | 🟡 中等 |
| **功能暴露过度** | 策略、联网、模板、场景全部默认可见 | 🟡 中等 |
| **对话区杂乱** | 每条消息含角色/意图/模型/操作按钮 | 🔴 严重 |
| **侧边栏臃肿** | 三页切换（对话/模板/工具），每页信息密集 | 🟡 中等 |

## 三、提议的新设计

### 设计原则

1. **输入框是上帝**——最大的视觉权重，最大的交互区域
2. **对话流干净**——每条消息只有内容。元信息折叠到悬停/点击时出现
3. **控件藏起来**——只有"当前在用什么"可见，切换操作藏进弹出菜单
4. **Agent 操作可见**——Workflow 步骤 / Planner 进度是对话流的一部分，不额外占空间
5. **保留启文 DNA**——黑金配色保留，克制优雅。不变成 Claude/ChatGPT 的克隆

### 新布局

```
┌──────────────────────────────────────────────────┐
│  ✦ Polaris                     [侧栏] [设置] [✕] │  ← 一行顶栏
├──────────────────────────────────────────────────┤
│                                                  │
│  对话流（纯净版）                                   │
│                                                  │
│  用户: 帮我写一个 Python 排序算法                   │
│                                                  │
│  Polaris:                                        │
│  [Coder → Reviewer 工作流]  ← 折叠块，可展开       │
│                                                  │
│  def quicksort(arr):                             │
│    ...                                           │
│                                                  │
│  [复制] [重试]  ← 悬停时才出现                     │
│                                                  │
│  ──────────────────────────────────────────────── │
│                                                  │
│  ┌────────────────────────────────────────────┐  │
│  │ 向 Polaris 提问...                          │  │  ← 输入框最大面积
│  │                                            │  │
│  │                                    [✦ ● ◈] │  │  ← 策略切换缩成图标
│  │                               [🎤] [↑ 发送] │  │
│  └────────────────────────────────────────────┘  │
│  优质模式 · DeepSeek V4 Flash · 按 / 使用命令    │  ← 一行状态提示
└──────────────────────────────────────────────────┘
```

### 具体改动

**顶栏：从 4 行 → 1 行**
- 当前：logo + token条 + 4 个功能按钮 + 3 个窗口按钮
- 改后：✦ Polaris [Cmd+P] ······ [侧栏☰] [设置⚙] [✕]
- Token 条移到侧边栏底部（不常用但要能看）
- 窗口控制保留（Electron 需要）

**输入区：从 3 行 → 1 个大卡片**
- 当前：模板行 + 语音/输入/发送行 + 策略/联网/模板行 + 提示行
- 改后：一个大输入卡片
  - 输入框占满宽度，高度自适应（最小 48px，最大 200px）
  - 右下角一行小图标：策略选择器（✦●◈）、联网开关、麦克风、发送
  - 按 `/` 弹出命令菜单（内置在输入框中）
  - 输入框底部一行极小文字："优质模式 · DeepSeek V4 · Enter 发送 · / 命令"

**侧边栏：精简为 2 页**
- 对话历史（页 1）：列表 + 搜索
- 模板库（页 2）：合并工具面板进去，工具显示在模板列表中
- Token 用量 + 设置入口放底部

**消息行：极简化**
- 当前每条消息显示：意图标签 + 箭头 + 模型名 + 正文 + 模型名（重复）+ 复制/重试/分支/编辑 按钮
- 改后：
  - 正常状态：只显示气泡
  - 悬停气泡：左下角淡出"Claude Sonnet 4 · 2.3s"
  - 悬停气泡：右下角淡出操作按钮（复制/重试）
  - 意图和路由信息：折叠在消息顶部的可展开小横条里

**Agent 操作流：Workflow 步骤可视化**
- 不再是文字"✓ code: coder — 代码已生成..."
- 而是小型步骤卡片：`[拆解 → 编码 ✓ → 审查中... → 输出]`
- 每个步骤带状态图标和进度

**Planner / Intervention 卡片：保留但缩小**
- 当前：全宽大卡片
- 改后：内联通知条，带确认/忽略按钮，2 行高度

## 四、讨论：需要你拍板的 4 个设计决策

### Q1：窗口模式

**A**：保持当前全窗口（1100×750），适合长时间深度工作  
**B**：改为浮动小窗（600×500），类似 Raycast，Cmd+Shift+Space 唤起即用即走  
**C**：两种都支持——默认大窗，支持最小化到小窗模式

### Q2：侧边栏

**A**：保留侧边栏，但精简（当前 3 页 → 2 页）  
**B**：去掉侧边栏，对话列表移到下拉菜单（极致精简，像 ChatGPT Desktop）  
**C**：侧边栏默认收起，需要时滑出（像 Notion）

### Q3：消息元信息的展示方式

**A**：悬停才显示（最干净）  
**B**：始终显示但极淡（透明度 0.3）  
**C**：折叠在消息顶部的可展开横条里

### Q4：是否保留当前启文风格的整体框架

**A**：保留。在黑金配色的框架内做减法  
**B**：推倒重来。完全参考 Claude Desktop 的极简风格
